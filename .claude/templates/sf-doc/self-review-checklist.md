## 生成前セルフレビューチェックリスト

`generate_basic_doc.py` / `generate.py` を **呼ぶ前に**以下を全件確認する。sf-doc-overview-writer / sf-doc-objects-writer 共用。問題があれば修正してから生成へ進む。

> sf-doc の xlsx は docs/ または組織接続から決定論的に生成されるため、レビュー対象は生成物ではなく **writer 自身の入力判断**（出力先・バージョン方式・新規/更新判定・対象・接続先）である。JSON 系 writer（sf-design 等）のような機械検証スクリプト（`check_*.py`）は sf-doc には存在しない。本チェックリストのみで生成前ゲートとする。

### 共通チェック（両 writer）

- [ ] **出力先の妥当性**: `{output_dir}/01_基本設計/` が正しい出力先か（誤った `output_dir` を渡していないか）
- [ ] **version_increment の妥当性**: `minor`/`major` の判断が実態と一致しているか（項目追加程度なら `minor`、オブジェクト構成の大幅変更等なら `major`）
- [ ] **新規/更新判定と `--source-file` の整合**: 既存 xlsx が存在するのに新規モード（`--source-file` なし）で実行しようとしていないか、逆に存在しないのに更新モードを指定していないか
- [ ] **author の妥当性**: `{author}` が空でない・意図した作成者名になっているか

### 概要書（overview）固有チェック

- [ ] **docs/ 情報源の充足確認**: `org-profile.md` / `requirements.md` / `system.json` / `_data-model.md` / `swimlanes.json` の存在確認結果を認識し、欠落分がプレースホルダー・空欄になる旨を理解した上で進めているか

### オブジェクト定義書（objects）固有チェック

- [ ] **対象オブジェクトの選択**: 選択したオブジェクトリストに漏れ・余剰がないか（ユーザーの指定と一致しているか）
- [ ] **接続先組織の妥当性**: `{SF_ALIAS}` が意図した接続先組織か（本番組織への誤接続がないか）
- [ ] **システム名称の妥当性**: `{システム名称}` が正しく設定されているか

### チェック後

全項目を確認し問題がなければ生成スクリプトを実行する。生成後の検証は各 writer 既存の存在・mtime・size 確認（Phase 2 / Phase 7-2 の post-check）で担保する。
