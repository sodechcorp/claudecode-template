# option-similar-future-prevention

## 何をするか

今回の課題と同種の問題が再発しないための仕組み・ルール化を提案する。個人ミスへの対処ではなく、構造的に防ぐアプローチを提案する。

## 実行手順

1. 今回の根本原因を確認する（causal-chain-analysis の結果）
2. 根本原因が「構造的問題」かどうかを評価する:
   - **構造的問題** 例: 「設計上の制約が知られていないため同じミスが繰り返される」「テストが不足しているため変更時に気づけない」
   - **個人ミス** 例: 「設定値の入力ミス」「一時的なデータ問題」→ 再発防止策は不要

3. 構造的問題の場合、以下の観点で再発防止策を提案する:

   **コード・設計レベル**:
   - バリデーション・ガードを追加する
   - 共通ユーティリティで制約を強制する
   - コードコメントで「やってはいけないこと」を明示する

   **テストレベル**:
   - 今回のバグを再現するテストケースを追加する（リグレッション防止）
   - edge case のテストカバレッジを強化する

   **ドキュメントレベル**:
   - docs/decisions.md に「この制約は〇〇の理由で必要」と記録する
   - 開発ガイドラインに注意事項を追加する

4. 採用した再発防止策を実施する（コード修正 / docs 更新）
