# option-multi-cause-hypothesis

## 何をするか

原因仮説を 3 件以上立てて尤度比較する。単一原因への思考ロック（最初の仮説に固執する認知バイアス）を防ぐ。

## 実行手順

1. 現時点で最も疑っている仮説（仮説 A）を 1 文で言語化する
2. 仮説 A とは異なる視点から仮説 B・C を立てる（最低 2 個追加、合計 3 個以上）:
   - データ問題（レコードの状態・フラグ値が原因）
   - 権限問題（FLS / プロファイル / OWD が原因）
   - タイミング問題（非同期処理・トリガー順序が原因）
   - 設定問題（カスタム設定・カスタムメタデータが原因）
   - コードロジック問題（条件分岐・演算が原因）
   - 実行時挙動問題（コード静的読解では確認できず、Apex デバッグログや SOQL 実値確認が必要な仮説は option-apex-debug-log / option-cross-record-comparison に明示的に渡す）
3. 各仮説について以下を評価する:
   - 根拠（どの証拠がこの仮説を支持するか）
   - 反証（どの証拠がこの仮説に矛盾するか）
   - 推定尤度: 高 / 中 / 低
4. 尤度の高い仮説から検証する順序を決める
