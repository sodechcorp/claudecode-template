# option-release-note-generation

## 何をするか

変更内容・影響範囲・注意事項をまとめたリリースノートを生成する。関係者への周知・ドキュメントとして活用する。

## 実行手順

1. 以下の情報を収集する:
   - 課題 ID・件名
   - 変更の概要（業務ユーザーが理解できる言葉で）
   - 影響を受けるユーザー・機能
   - 変更の有効日時
   - 操作変更がある場合: 変更前後の操作手順

2. リリースノートを以下のフォーマットで生成する:

   ```
   # リリースノート: {件名}
   リリース日: {YYYY-MM-DD}
   課題 ID: {issueID}

   ## 変更内容
   {業務ユーザーが理解できる言葉で変更を説明する}

   ## 影響範囲
   - 対象ユーザー: {プロファイル名・部署名}
   - 対象機能: {機能名・画面名}

   ## 操作変更（ある場合）
   - 変更前: {以前の操作手順}
   - 変更後: {新しい操作手順}

   ## 注意事項
   {特記事項・注意点}
   ```

3. リリースノートを `docs/logs/{issueID}/release-note.md` に保存する
4. 関係者への周知方法をユーザーに提案する（Backlog コメント・Slack 等の案を提示するのみ）。**Claude は Backlog コメント等を投稿しない**。投稿・送信が必要ならユーザーが手動で実施する。Claude はリリースノート本文の起案までを担当する

## 出力

`docs/logs/{issueID}/release-note.md` を新規作成して保存する。
