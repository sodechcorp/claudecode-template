# option-stakeholder-notification

## 何をするか

関係者への完了通知を実施する。Backlog コメント・Slack・メール等での周知を行う。

## Claude の責務範囲

**Claude は Backlog コメント・お客様向けメッセージを投稿しない**。文面案を **テキストで提示するのみ**。投稿・送信は人間が判断して手動で実施する。`mcp__backlog__add_issue_comment` 等の Backlog 書き込み系 MCP は使わない（社内向け・社外向けを問わず）。

## 実行手順

1. 通知先を確認する:
   - 課題の依頼者（Backlog 課題作成者）
   - コメントで要件を追加した関係者
   - 影響を受けるユーザーの担当者・マネージャー
   - 他チームの担当者（横断影響がある場合）

2. 通知内容を準備する（業務ユーザー向け・技術的な詳細は不要）:
   ```
   件名: 【対応完了】{課題件名}

   お疲れ様です。

   ご依頼いただきました「{課題件名の要約}」の対応が完了しました。

   ■ 対応内容
   {変更の概要を業務的に説明}

   ■ 有効日時
   {リリース日時}

   ■ 確認方法
   {どの画面・操作で確認できるか}

   ご不明点があればお知らせください。
   ```

3. 通知方法別に **文面案をテキストで提示する**（Claude は投稿・送信しない。ユーザーが手動で実施）:
   - **Backlog コメント**: 起案文をチャットで提示。ユーザーが Backlog UI から手動で投稿
   - **Slack / メール**: 起案文をチャットで提示。ユーザーが手動で送信（必要なら communicator エージェントに依頼してトーン調整）

4. 通知文案を test-report.md に記録する

## 出力

test-report.md に追記:

## ステークホルダー通知

- 通知先: {氏名 / グループ名}
- 通知方法（案）: Backlog コメント / Slack / メール
- 通知文案: （上記 Step 3 で提示した文面）
- 対応: ユーザーに文面案を提示済み（投稿・送信はユーザーが実施）
