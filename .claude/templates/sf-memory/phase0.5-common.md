# Phase 0.5 規約適合チェック（共通テンプレート）

各カテゴリは差分更新モード開始時にこのテンプレートを実行する。
各エージェントは「## カテゴリ固有の必須項目リスト」のみを自ファイルに記述し、共通手順はここを参照する。

---

## 共通手順

1. 対象ディレクトリの既存 `.md` ファイルを Glob で列挙する
2. 各ファイルの先頭 80 行を Read して H2 見出し（`## `）を抽出する
3. カテゴリ固有の必須項目リスト（各エージェントファイルに記載）と照合する
4. 欠落項目があれば「補完アクション」として記録し、対象ファイルへの補完内容をプランニングする
5. 古いスキーマ（必須 H2 が欠落しているもの）は Phase 生成フェーズで補完する（手動追記保持）

**ルール**:
- 規約適合補完は手動追記より優先する（cat1 の方針に統一）
- 存在しない項目は追加してよい。既存の手動追記・設計コメントは削除しない
- 補完で追加する新規セクションは末尾に追記する（既存構造を破壊しない）

---

## カテゴリ別 必須 H2 チェックリスト

各エージェントは自ファイルにカテゴリ固有の必須項目を記述し、本ファイルを参照する。

### cat2: catalog/*.md
```
必須 H2: 基本情報 / 項目一覧 / リレーション / 権限マトリクス / 自動化・ビジネスルール / 被参照（被 Lookup）
```

### cat3: data/*.md
```
必須 H2（master-data.md）: マスタ一覧 / レコード件数 / 代表値サンプル
必須 H2（automation-config.md）: 承認プロセス / キュー割り当て / 自動化設定
必須 H2（email-templates.md）: テンプレート一覧 / 差し込み項目
```

### cat4-*: design/{type}/*.md
```
必須フロントマター: CMP-ID / FR-XXX / 関連UC / 担当オブジェクト / 改版履歴
```

### cat5: feature_groups.yml
```
必須キー: generated_at / groups（dict 形式）
グループ毎の必須フィールド: id / name / actor / uc_anchors / components / assignment_confidence
```
