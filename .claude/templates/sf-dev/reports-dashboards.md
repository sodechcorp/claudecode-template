# レポート / ダッシュボード（Reports / Dashboards）

## レポート作成時

1. `docs/data/reports-dashboards.md` で類似レポートが既にないか確認
2. **レポートタイプ（ReportType）選定**: 対象オブジェクトの組合せを決める。標準レポートタイプ（例: 商談、ケース、キャンペーンとキャンペーンメンバー）またはカスタムレポートタイプのどれが適切か、**組織の Setup > レポートタイプ一覧で実在を確認してから選択する**。目的に合うレポートタイプが存在しない場合はカスタムレポートタイプの作成が必要
3. **形式（Format）選定**: 集計が必要か（サマリー / マトリックス）、詳細一覧か（表形式）、複数レポートの結合が必要か（結合）を確認

**確認事項:**
- フィルター条件: 日付範囲・担当者・レコードタイプ等
- グルーピング: サマリー/マトリックスの場合、何で集計するか
- 保存先フォルダ: 公開 / 個人 / チーム
- ダッシュボードに追加するか

**注意:** クロスフィルター・バケット項目・カスタムサマリー数式は複雑になりがちなので要件を再確認

**⚠️ 実在確認ガード**: レポートタイプ名・グルーピング可能な項目・サマリー数式（`PARENTGROUPVAL` / `PREVGROUPVAL` 等）は組織のレポートタイプ定義・UI に依存する。**記憶で名前を断定せず、組織の Setup またはレポートビルダーで実在を確認してから案内する**。確認できないものは `[要確認]` を付け、確定した事実として提示しない

## ダッシュボード作成時

- ダッシュボードの閲覧者（実行ユーザー）を確認: 「閲覧者のデータ」vs「固定ユーザーのデータ」
- 自動更新頻度の確認（毎日/毎週/手動）
