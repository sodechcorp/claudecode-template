# カテゴリ2 オブジェクト定義書 テンプレート

---

## オブジェクト定義書の必須構成

各オブジェクトに対して `docs/catalog/{standard|custom}/<オブジェクト名>.md` を以下の構成で生成する。

```markdown
# {オブジェクト名}（{API名}）

## 基本情報
| 項目 | 値 |
|---|---|
| オブジェクト種別 | カスタム / 標準 |
| 用途 | （業務的な用途を具体的に記述） |
| レコード件数 | {件数} |
| 関連UC | UC-XX: {UC名}、UC-XX: {UC名} |
| 関連FR要件 | FR-XXX、FR-XXX |

## リレーション
（Mermaid ER図を含める。このオブジェクトを中心に、親・子・参照先を全て図示）

## 全項目一覧
（標準項目・カスタム項目を分けて全量記述。
  型・必須/任意・ユニーク・デフォルト値・用途を列として持つ）

## ピックリスト値
（ピックリスト項目ごとに全値を列挙）

## 入力規則
（名前・条件（数式全文）・エラーメッセージ・有効/無効）

## 自動化（このオブジェクトに関連するApex/Flow）
（どのApexクラス・Flow・トリガーがこのオブジェクトを操作するか）

## 権限マトリクス
（プロファイル/権限セット別の Read/Create/Edit/Delete 権限）

## 所見・注意点
```

## cross-reference の記載（重要）

「このオブジェクトがどのUCで使われるか」「どのApex/FlowがSOQL/DMLで操作するか」を必ず記載する。情報がない場合は `**[要確認]**` を入れる。

---

## 備考列マーカー規約

「全項目一覧」テーブルの備考列に使用するマーカー。sf-memory-quality.md のマーカー文化と統一する。

| マーカー | 意味 | 付与タイミング |
|---|---|---|
| `**[本番デプロイ待ち]**` | Sandbox/dev 組織に存在するが本番組織に未反映の項目 | Phase 1.5 で本番突合モードを選択し、Phase 2.6 の差分計算で検出された場合 |
| `**[推定]**` | メタデータに明記がなく、用途・意味を推測した箇所 | describe 結果から業務的意味を補間した場合 |
| `**[要確認]**` | 情報が取得できなかった・意図が不明な箇所 | describe 失敗・設計者確認が必要な場合 |

**記載例**（備考列が空の場合）:
```
| IsCriminal__c | 反社チェックフラグ | チェックボックス | | **[本番デプロイ待ち]** |
```

**記載例**（備考列に既存の説明がある場合）:
```
| Status__c | ステータス | テキスト | ○ | **[本番デプロイ待ち]** GF-341 対応項目 |
```

> テーブル列構成（API 名 / 表示ラベル / 型 / 必須 / 備考）は変更しない。マーカーは備考列の先頭に置く。

## `_index.md` 列規約（下流パーサー連携）

`_index.md` のオブジェクト一覧テーブルは以下のキーワードをヘッダ行に含めること（列順は問わない）:

| キーワード | 必須 | 用途 |
|---|---|---|
| `API名` | 必須 | 下流 `generate_basic_doc.py` が識別キーとして読む |
| `ラベル` | 必須 | 日本語表示名。ER図のノード表示に使用 |
| `種別` | 推奨 | `カスタム` / `標準` |

---

## Mermaid ERD 記法の規約（下流パーサー連携・厳守）

`generate_basic_doc.py` が正規表現で関係線を抽出するため以下を守ること。

**関係線の構文**:
```
ParentApiName ||--o{ ChildApiName : "fieldLabel"
```

- 実線 `--` に統一（点線 `..` も受理されるが原則実線）
- 右端カーディナリティ: `o{` / `|{` / `||` / `o|` のいずれか
- ラベル部分（`:` の右）は **FK項目の API名 または日本語ラベルを必ずダブルクォートで囲む**（空文字列は不可）
- **標準オブジェクトも含める**: Account / Contact / Opportunity / Case / Lead 等、カスタムから参照されているものは必ずノードに含める

**悪い例**（関係線がマッチしない）:
```
Account -- ContractApplication__c
Account o-- Contract
```

**良い例**:
```
Account ||--o{ ContractApplication__c : "AccountId__c"
Opportunity ||--o{ ContractApplication__c : "OpportunityId__c"
```
