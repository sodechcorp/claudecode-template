# インラインスクリプト実行ルール（`python -c` 等・厳守）

## 背景

Bash ツール引数（`command`）に改行・インデント・多重クォートを含む多行スクリプトを埋め込むと、tool-call 自体の JSON 生成/パースが壊れて処理が停止するリスクがある。特に `if` / `try-except` / `for` / `while` などインデントを伴う制御構文は危険度が高い（SNM-14 で実際に発生・再現確認済み）。

## ルール

1. **`python -c "..."` は単一物理行に限定する**。`;` 区切り・改行なしで完結する範囲のみ許可する。
2. **ループ・try-except・if 分岐など多行ロジックが必要な場合は、単一行化を無理に試みず `{tmp_dir}` に `.py` を Write → `python {tmp_dir}/xxx.py` で実行する**（一時ファイルの配置ルールは [tmp-file-rules.md](tmp-file-rules.md) 参照）。単純な条件分岐は三項演算子（`A if cond else B`）で単一行に畳み込めないか先に検討してよいが、可読性を損なうほど複雑になるなら Write に切り替える。
3. **`jq` は使用しない**（本テンプレート運用環境では未インストール確認済み）。JSON 整形が必要な場合は `sf data query --result-format human|csv` 等でそもそも JSON パースを避けることを優先する。python を使わざるを得ない場合も上記 1-2 のルールに従う。

## 判定表

| スクリプトの内容 | 対処 |
|---|---|
| 単一の式・単純な import/print の羅列（`;` 区切りで表現できる） | `python -c "a; b; c"` のまま単一行で書く |
| `if` / `try-except` / `for` / `while` を含む | 単一行の三項演算子で表現できるかまず検討。無理なら `.py` を Write して実行する |
| 複数行文字列・ヒアドキュメントで組み立てる | 同上（`.py` Write） |

## 実行前セルフチェック（必須）

Bash ツールに `python -c` を渡す直前に、その文字列に改行文字が含まれていないか目視確認する。含まれていたら本ルール違反 → 単一行化するか `.py` Write に切り替える。
