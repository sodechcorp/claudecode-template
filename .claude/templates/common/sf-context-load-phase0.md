# sf-context-loader 呼び出し仕様（Phase 0 共通）

## 概要

`sf-context-loader` は `docs/` 配下のファイルを横断検索し、タスクに関連する業務コンテキスト（オブジェクト定義・UC仕様・設計意図・注意点）の要約を返す共通エージェント。

## 呼び出しパラメータ

| パラメータ | 説明 | 必須 |
|---|---|---|
| `task_description` | ユーザー指示またはタスクの概要（「何をするか」を日本語で記述） | 必須 |
| `project_dir` | プロジェクトルートパス（`docs/` の親ディレクトリ）。不明な場合はカレントディレクトリを指定 | 必須 |
| `focus_hints` | 対象を絞り込むキーワード（オブジェクト名・機能名等）の配列。空配列 `[]` で docs/ 全体を検索 | 任意 |

## `focus_hints` の記法（統一ルール）

`focus_hints` は必ず **ダブルクォート付き JSON 文字列配列** で記述する。次の 3 形式のいずれか:

| 用途 | 記法 | 例 |
|---|---|---|
| 絞り込み不要（docs/ 全体検索） | 空配列 | `focus_hints: []` |
| 固定キーワードで絞り込む | 文字列配列 | `focus_hints: ["callout", "named_credential"]` |
| タスクから動的抽出（プレースホルダ） | 文字列1要素の配列 | `focus_hints: ["{F-番号・オブジェクト名等を抽出して列挙}"]` |

- **クォート必須**: `[callout, api]` のようなクォートなし記法は禁止
- `"{...}"` 形式のプレースホルダは、Claude が sf-context-loader 呼び出し前に実値へテキスト置換（`.claude/CLAUDE.md` §テンプレート置換ルール準拠）。置換後も文字列配列の形を保つ
- 予約値 `"knowledge-only"` を含めると knowledge-only モードに切り替わる（backlog-investigator / backlog-planner 専用。通常エージェントでは使用しない）

## 結果の標準解釈

### ✅ 関連コンテキストが返った場合

返された内容（関連オブジェクト・UC・ビジネスルール・注意点）を以降の判断材料として保持する。

### ⚠️ 「該当コンテキストなし」が返った場合

`docs/` 未整備または task と docs が無関係。ただし **最低限 `docs/_README.md` を 1 回 Read する**（存在する場合のみ）。ドキュメント体系・用語集の所在だけ把握してから次の手順へ進む。

**不確実マーカー運用ルール**（必ず守ること）:

1. プロジェクトの**構成・項目名・自動化・業務フロー**に関する具体的な事実を回答に含める場合、必ず `[組織未調査]` または `[資料未確認]` マーカーを付ける
2. 推測・一般論・記憶ベースの言及には `[推定]` を付ける
3. 人間に確認しないと決められない事項には `[要確認]` を付けてユーザーに質問する
4. **マーカーなしの断定は禁止**（`step-0c-template.md` §実装裏付けルール参照）

マーカーの詳細定義: [`uncertainty-marker-spec.md`](./uncertainty-marker-spec.md)

### ⚠️ エラー / タイムアウトが発生した場合

`sf-context-loader` の呼び出し自体が失敗した場合も、**最低限 `docs/_README.md` + `docs/overview/org-profile.md`（用語集）を直接 Read する**（各ファイルが存在する場合のみ）。それ以降は「該当コンテキストなし」と同じマーカー運用を適用する。
