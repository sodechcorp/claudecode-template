# uncertainty-marker-spec — 不確実マーカー定義

回答・成果物に含む不確実マーカーの正規定義。各エージェント・メインスレッドで統一して使う。

---

## マーカー一覧

| マーカー | 使いどころ | 使用エージェント |
|---|---|---|
| `[推定]` | 記憶・推論で回答しているが、実装・資料確認が取れていない | 全エージェント |
| `[要確認]` | 人間（ユーザーまたはチーム）の確認が必要。Claude が答えを持っていない | 全エージェント |
| `[出典不明]` | 引用・出典帰属を確認できなかった | 全エージェント |
| `[資料未確認]` | sf-memory 収集フェーズで参照すべき資料が見つからない | sf-analyst 系のみ |
| `[組織未調査]` | org-profile.md 等の組織調査が未完了 | sf-analyst 系のみ |

---

## 判断フロー（断定 vs 推定 vs 要確認）

```
断定してよいか？
├─ YES（実装コード Read 済み or 自明な仕様）→ そのまま回答
├─ YES だが資料確認してない → [推定] を付けて回答
└─ NO（人間に聞かないと決まらない）→ [要確認] を付けて質問提示
```

**「断定してよい」条件**（いずれかを満たす）:
- 対象の Apex/Flow/field-meta.xml/permissionset を Read して確認した
- Salesforce 公式仕様として自明（例: レコードの主キーは Id フィールド）
- docs/ に明記されている

**「断定してはいけない」条件**（どれかに該当する）:
- 実装ファイルを Read していない状態で挙動を断言
- プロジェクト固有の業務ルール・命名・経緯を記憶で答えている
- 「たぶん」「おそらく」「一般的には」で始める回答

---

## 禁止パターン

| 禁止 | 代替 |
|---|---|
| 「○○の場合は△△になります」（実装未確認） | 「[推定] ○○の場合は△△になると思われます。実装を確認します」 |
| 「〜さんがコメントしていました」（Backlog 未確認） | 「[出典不明] Backlog コメントを再取得して確認します」 |
| 「たぶん〜でしょう」で断定 | 「[推定] 〜の可能性がありますが、確認が必要です」 |
| 「わかりません」で終わる | 「[要確認] 〜が不明です。docs/ に記載がなく、〜さんへの確認を推奨します」 |
