# answer-scope-spec — ユーザー回答時のスコープ管理

各エージェントが質問・依頼に回答する際のスコープ管理ルール。
CLAUDE.md §「ユーザー回答時のスコープ管理」に要旨を掲載している。本ファイルは詳細・適用例・追加ルール記入欄を提供する。

---

## 1. 回答の構造

1. **質問に直接答える1行を最初に書く** — 冒頭に結論を置く。背景・理由は2行目以降
2. **補足は最大3項目まで** — 関連情報の羅列は禁止。重要度の高いもの3件に絞る
3. **派生事項は明示的に分離する**:
   ```markdown
   ## 派生事項（質問外）
   - 〜について補足があります（対応しますか？）
   ```
   派生事項として分離しない限り、質問外の情報を本文中に混入しない

---

## 2. 禁止パターン

| 禁止 | 理由 |
|---|---|
| 質問されていないコード書き換え・リファクタを提案する | 作業範囲の勝手な拡大 |
| 「ついでに〜も直しておきました」系の追加作業を無断実施 | 承認なし変更 |
| 仕様確認依頼に対して「他にも確認した方が良いこと」を本文に列挙する | スコープ外助言 |
| 「〜についても検討が必要です」を本文中に羅列する | 派生事項セクションへ移動 |
| ユーザーの質問に答える前に前提確認・追加質問を先に出す | 結論後退 |

---

## 3. 「ついでに」処理の手順

追加作業が適切と判断される場合:

1. 本来の依頼を完了する
2. **「## 派生事項（質問外）」セクションで明示提案する**:
   ```markdown
   ## 派生事項（質問外）
   〜についても変更が必要と思われます。対応しますか？
   ```
3. ユーザーの承認を得てから実施する

---

## 4. 適用例

**NG（スコープ外実施）**:
> Q: Account の SOQL クエリを最適化して
> A: クエリを最適化しました。また Opportunity の SOQL も最適化した方がよさそうなので変更しておきました。〜

**OK（派生事項として分離）**:
> Q: Account の SOQL クエリを最適化して
> A: Account の SOQL を最適化しました。[コード]
>
> ## 派生事項（質問外）
> Opportunity の SOQL も同様のボトルネックがあります。最適化しますか？
