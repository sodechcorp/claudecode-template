# 新規メタデータ作成時 権限・基本設定チェックリスト

> **目的**: 新規カスタムメタデータを作成した際に権限・基本設定を必ず設定し、漏れを防ぐ。
> **原則**: 各項目は「振った/振らない」を判断ではなく**必ずチェックして明示する**（漏れの黙殺禁止）。
> **権限ポリシー**: 標準プロファイルは編集禁止。付与先は必ず**権限セット**（`PREFIX_` プレフィックス）で対応する。

---

## カスタム項目（CustomField）を作成したとき

- [ ] **FLS（項目レベルセキュリティ）**: 対象権限セットの `fieldPermissions` に `readable: true` / `editable: true` を追加
  - 参照のみでよい項目は `editable: false` / 編集不要の場合は意図を明示する
- [ ] **ページレイアウト配置**: 対象オブジェクトの `.layout-meta.xml` に追加（項目が画面に表示されること）
  - 複数レイアウトがある場合は全レイアウトにおける表示要否を判断する
- [ ] **コンパクトレイアウト**: リストビュー・関連リスト・モバイルで表示すべき重要項目かどうかを確認
- [ ] **検索レイアウト**: 検索結果に表示すべき項目かどうかを確認
- [ ] **必須/読み取り専用**: プロファイル/権限セットレベルの必須・読み取り専用設定が要件に合っているか確認

---

## カスタムオブジェクト（CustomObject）を作成したとき

- [ ] **CRUD権限**: 対象権限セットの `objectPermissions` に `Create/Read/Edit/Delete` を必要な範囲で付与
  - 削除権限は慎重に設定（不要な場合は `allowDelete: false`）
- [ ] **タブ（CustomTab）**: オブジェクトにタブが必要か判断し、必要なら `.tab-meta.xml` を作成
  - 対象権限セットの `tabSettings` に `visibility: Available`（または `DefaultOn`）を設定
  - タブが表示されるアプリ（Lightning App）への追加を確認
- [ ] **ページレイアウト**: デフォルトレイアウト（`-meta.xml`）が存在し、`ProfileLayout` に割り当てられているか確認
- [ ] **レコードタイプ**: 必要かどうか判断（不要ならマスターレコードタイプのみ）
  - 必要な場合は下記「レコードタイプ」セクションも実施
- [ ] **アプリへの追加**: 利用者が使うアプリの nav items にオブジェクトが含まれているか確認

---

## レコードタイプ（RecordType）を作成したとき

- [ ] **プロファイル/権限セットへの割当**: 対象権限セットの `recordTypeVisibilities` に追加し、`visible: true` を設定
  - デフォルトにするかどうかを明示（`default: true/false`）
- [ ] **ページレイアウト割当**: `ProfileLayout`（`recordTypeAssignments` または `profileActionOverrides`）に新レコードタイプ → レイアウトのマッピングを追加
- [ ] **ピックリスト値割当**: レコードタイプに紐づくピックリスト項目の使用可能値を制限する場合は `RecordTypePicklistValues` を設定
- [ ] **デフォルト設定**: 複数レコードタイプがある場合、作成時のデフォルトが適切か確認

---

## タブ（CustomTab）を作成したとき

- [ ] **権限セットのタブ表示設定**: `tabSettings` に `visibility: Available` / `DefaultOn` / `Hidden` を設定
- [ ] **アプリへの追加**: 対象 Lightning App の `navItems` にタブを追加
- [ ] **モバイルナビ**: モバイルアプリで表示すべき場合は Mobile Navigation メニューへの追加を確認

---

## Apex クラス / Visualforce ページを作成したとき

- [ ] **Apex クラスアクセス（権限セット）**: `classAccesses` に `enabled: true` を追加
  - 全ユーザーに不要なクラス（バッチ・スケジューラ等の内部実行のみ）は不要だが、その意図を明示する
- [ ] **VF ページアクセス（権限セット）**: `pageAccesses` に `enabled: true` を追加

---

## フロー（Flow）を作成したとき

- [ ] **実行権限**: 画面フロー（Screen Flow）はユーザーが実行できるか確認（権限セットまたはプロファイルの「フロー実行」を確認）
- [ ] **埋め込み先の権限**: LWC / フレックスページに埋め込む場合、そのコンポーネント/ページへのアクセス権限も確認
- [ ] **Process Automation 設定**: 組織の自動化設定（Process Automation Settings）が要件に影響しないか確認

---

## 共通（種別問わず、新規作成後に必ず確認）

- [ ] デプロイ前に `sf project deploy start --dry-run` で問題がないことを確認
- [ ] 権限セット・ページレイアウト・タブを変更した場合は `docs/catalog/` の定義書を更新
- [ ] 上記を全チェックしたことを作業完了コメントに明記する（チェック漏れの黙殺禁止）
