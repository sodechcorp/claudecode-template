# 新規メタデータ作成時 権限・基本設定チェックリスト

> **目的**: 新規カスタムメタデータを作成した際に権限・基本設定を必ず設定し、漏れを防ぐ。
> **付与先の原則**:
> - **システム管理者プロファイル**: 必ず全権限を付与（必須・例外なし）
> - **その他のプロファイル**: 自動付与禁止。組織の権限設計（プロファイル中心 or 権限セット中心）を確認し、ユーザーと相談して付与先を決める。未検討・黙殺は禁止
> - **権限セット（PermissionSet）**: 自動で作成・更新しない。ユーザーが明示指定したときのみ対応
> **原則**: 各項目は「実施した/不要と判断した/ユーザーに確認中」を必ず明示する（黙殺禁止）。

---

## カスタム項目（CustomField）を作成したとき

- [ ] **FLS（項目レベルセキュリティ）— システム管理者**: `fieldPermissions` に `readable: true / editable: true` を追加（必須）
- [ ] **FLS — その他プロファイル**: 付与先をユーザーと確認してから設定する（自動付与禁止）
  - `readable: true`（参照許可）
  - `editable: true`（編集許可）— 読み取り専用でよい場合は `false` とし、その意図を明示する
- [ ] **ページレイアウト配置**: 対象オブジェクトの `.layout-meta.xml` に追加（項目が画面に表示されること）
  - 複数レイアウトがある場合は全レイアウトにおける表示要否を判断する
- [ ] **コンパクトレイアウト**: リストビュー・関連リスト・モバイルで表示すべき重要項目かどうかを確認
- [ ] **検索レイアウト**: 検索結果に表示すべき項目かどうかを確認

---

## カスタムオブジェクト（CustomObject）を作成したとき

- [ ] **オブジェクト権限（CRUD）— システム管理者**: `objectPermissions` に追加（必須）
  - `allowCreate / allowRead / allowEdit / allowDelete` を要件に応じて設定
  - 削除権限は慎重に判断（不要なら `allowDelete: false`）
- [ ] **オブジェクト権限（CRUD）— その他プロファイル**: 付与先をユーザーと確認してから設定する（自動付与禁止）
- [ ] **タブ（CustomTab）**: オブジェクトにタブが必要か判断し、必要なら `.tab-meta.xml` を作成
  - システム管理者プロファイルの `tabVisibilities` に `DefaultOn` または `Available` を設定（必須）
  - その他プロファイルへのタブ表示設定は付与先をユーザーと確認してから設定する（自動付与禁止）
  - タブが表示されるアプリ（Lightning App）への追加を確認
- [ ] **ページレイアウト**: デフォルトレイアウト（`.layout-meta.xml`）が存在し、プロファイルに割り当てられているか確認
- [ ] **レコードタイプ**: 必要かどうか判断（不要ならマスターレコードタイプのみ）
  - 必要な場合は下記「レコードタイプ」セクションも実施
- [ ] **アプリへの追加**: 利用者が使うアプリの `navItems` にオブジェクト/タブが含まれているか確認

---

## レコードタイプ（RecordType）を作成したとき

- [ ] **プロファイルへの割当 — システム管理者**: `recordTypeVisibilities` に追加（必須）
  - `visible: true`
  - `default: true/false`（デフォルトにするかどうかを明示）
- [ ] **プロファイルへの割当 — その他プロファイル**: 付与先をユーザーと確認してから設定する（自動付与禁止）
- [ ] **ページレイアウト割当**: 各プロファイルのレイアウト割当（`.profile-meta.xml` の `layoutAssignments`）に新レコードタイプ → レイアウトのマッピングを追加
- [ ] **ピックリスト値割当**: レコードタイプに紐づくピックリスト項目の使用可能値を制限する場合は `RecordTypePicklistValues` を設定
- [ ] **デフォルト設定**: 複数レコードタイプがある場合、作成時のデフォルトが適切か確認

---

## タブ（CustomTab）を作成したとき

- [ ] **タブ表示設定 — システム管理者**: `tabVisibilities` に `DefaultOn` / `Available` / `Hidden` を設定（必須）
- [ ] **タブ表示設定 — その他プロファイル**: 付与先をユーザーと確認してから設定する（自動付与禁止）
- [ ] **アプリへの追加**: 対象 Lightning App の `navItems` にタブを追加
- [ ] **モバイルナビ**: モバイルアプリで表示すべき場合は Mobile Navigation メニューへの追加を確認

---

## Apex クラス / Visualforce ページを作成したとき

- [ ] **Apex クラスアクセス — システム管理者**: `classAccesses` に `enabled: true` を追加（必須）
  - バッチ・スケジューラ等の内部実行のみで直接ユーザーアクセス不要の場合は不要だが、その旨を明示する
- [ ] **Apex クラスアクセス — その他プロファイル**: 付与先をユーザーと確認してから設定する（自動付与禁止）
- [ ] **VF ページアクセス — システム管理者**: `pageAccesses` に `enabled: true` を追加（必須）
- [ ] **VF ページアクセス — その他プロファイル**: 付与先をユーザーと確認してから設定する（自動付与禁止）

---

## フロー（Flow）を作成したとき

- [ ] **実行権限**: 画面フロー（Screen Flow）はユーザーが実行できるか確認（プロファイルまたは権限セットの「フロー実行」を確認）
- [ ] **埋め込み先の権限**: LWC / フレックスページに埋め込む場合、そのコンポーネント/ページへのアクセス権限も確認
- [ ] **Process Automation 設定**: 組織の自動化設定が要件に影響しないか確認

---

## 共通（種別問わず、新規作成後に必ず確認）

- [ ] デプロイ前に `sf project deploy start --dry-run` で問題がないことを確認
- [ ] 権限・レイアウト・タブ変更した場合は `docs/catalog/` の定義書を更新
- [ ] 上記を全チェックしたことを作業完了コメントに明記する（漏れ・「不要と判断した」理由を含む）

---

## ⚠️ 権限セットについて

権限セット（PermissionSet）の作成・更新は**ユーザーが明示指定したときのみ**対応する。案件の役割・ロール設計に依存するため、自動では一切触らない。
