# cases/{案件キー}.md 出力スキーマ

`docs/knowledge/cases/<案件キー>.md`（案件詳細ファイル）の **出力構造の正本**。
セクション見出しの種類・順序・各節の意味を定義する。

> **参照方法**: backlog-releaser（/backlog Phase 3.8）および sf-analyst-cat6（/sf-memory Step 5）がこのスキーマを基準として生成する。抽出元データや経路固有フィールドは各エージェントが追加指定する。

---

## タイトル行

```
# <案件キー>: <件名>
```

- `<案件キー>` は Backlog の issueID / issueKey（例: GF-123）
- `<件名>` は Backlog 上の summary / 件名（短縮不可・原文ママ）

## ヘッダ行

```
生成日: {YYYY-MM-DD} / データソース: <経路固有の値>
種別: <issueType>
```

- `データソース` と `種別` の値は呼び出し側エージェントが指定する
- cat6（Backlog 由来）では追加で `実績工数: {actualHours}h` 行を挿入してよい
- 空行をはさんでから `## TL;DR` に進む

## セクション定義（順序固定）

以下の7節を必ずこの順序で出力する。節の省略・追加・順序変更は禁止。

| 見出し | 記述内容 |
|---|---|
| `## TL;DR` | 200字以内の1段落要約。何の課題で・何をして・どうなったかを端的に述べる |
| `## 症状・要件` | 課題の事実部分（現象・依頼内容）を整形。箇条書き推奨 |
| `## 調査・検討の経緯` | 検討の流れ・確認事項・判断分岐の時系列。推定の場合は `[推定]` を付与 |
| `## 採用方針` | 最終的に何を選んだか・選定理由 |
| `## 却下案・代替案` | 排除した案と理由。情報が取れない場合は省略可 |
| `## 教訓・再発防止` | 同様の問題を防ぐための知見。取れない場合は `[推定]` 付きで推測してよい |
| `## 関連リンク` | Backlog 検索 URL ほか。呼び出し側エージェントが追加リンクを差し込んでよい |

---

*このテンプレートを参照するエージェント: `backlog-releaser`（/backlog Phase 3.8）・`sf-analyst-cat6`（/sf-memory Step 5）*
