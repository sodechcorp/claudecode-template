# 確証なし時 / docs優先ルール（全エージェント・メインスレッド共通）

## 判断優先順位（鉄則）

`docs/` > Web 公式ドキュメント > 記憶

## 確証レベル別の行動

| 確証レベル | 行動 |
|---|---|
| 完全に確証あり（構文・自明な API 名・基本概念等の不変知識） | そのまま回答 |
| 8 割以上確証あり（記憶に従ってもよいが裏取りすべき） | 該当ファイル 1 つを Read して確認後回答 |
| 5 割以下、または最近の仕様変更がありそう | `docs/_README.md` Read → 該当ファイル Read → 不足なら Web 検索 → それでも不明なら「わかりません。〜を確認してください」 |
| プロジェクト固有事項（命名・業務ルール・組織設定・対応経緯） | **必ず `docs/` を見る。記憶で答えない** |

> SF系エージェント（sf-dev / sf-architect / qa-engineer / reviewer / data-manager / integration-dev）は、Phase 0 直後に [`.claude/templates/common/step-0c-template.md`](.claude/templates/common/step-0c-template.md) を Read して本ルール・CRITICAL 3 spec・不確実マーカーを一括確認する。

## メインスレッド直接応答ルール

ユーザーからの質問・依頼にメインスレッド（または assistant）が直接答える時:

1. **質問種別の判定**（業務系 / 技術系 / 作業依頼 / 雑談）
2. **業務系・技術系の場合**: `docs/_README.md` を先に Read して情報所在を把握する
3. **SF キーワード検知時** (`__c` / `CMP-` / `UC-` / `FR-` / オブジェクト名 / 「自動化」「権限」「ガバナ」等): **assistant に必ず委譲する**（メインスレッドで完結させない）
4. **[確証レベル別の行動]** に従う
5. **回答時**: 出典（参照ファイル名・行番号）を明示する
6. **わからない場合**: 推測で答えず「〜が確認できません」と明示する

雑談・コマンド呼び出し依頼・専門エージェント委譲が明確な場合はスキップ可。

## 業務理解 0 モード（「〜って何？」「〜の流れは？」系の質問）

1. `docs/_README.md` → 該当ファイル Read → 回答する
2. 業務用語が出てきたら `overview/org-profile.md` の Glossary で API 名対応を提示する
3. 過去の判断・採用方針が関わる場合は `docs/decisions.md` を引用する
4. 同類の過去案件があれば `docs/knowledge/case-index.md` を引用する
5. 既存資料に答えがない場合は「資料に記載がありません。〜さん（org-profile.md §キーパーソン一覧参照）に確認することを推奨します」と明示する

**絶対禁止**: 記憶でプロジェクト固有事項に答える / docs に書いていないことを「たぶん〜」と推測する / 「一般的には〜」と Salesforce 一般論にすり替える（プロジェクト固有事項の質問の場合）
